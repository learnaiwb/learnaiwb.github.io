http基础

form-data

http请求中，body为form-data模式，此时可通过form表达或者js的formdata对象进行构造数据，数据类型可以为文件或者键值对的模式，如下所示：

此时，Headers中的Content-Type就应该为Content-Type: multipart/form-data; boundary=—-WebKitFormBoundaryMHzissskK7K82uyt的格式，其中分隔符会应用到body中，如下所示：

在web开发中上传文件比较简单，一个普通的post表单再加上文件类型的标签就可以完成，上传的这些工作都是交给浏览器完成。但是在客户端上传文件时就需要自己写http上传相关的参数

x-www-form-urlencoded

将表单内的数据转换为键值对，如name=java&age=23;注意采用url编码格式，这意味着中文不会直接作为参数传入的。

raw

可以上传任意格式的文本，如text json xml html 等,下图列出了raw格式对应的html头文件。

Text text/plain
JavScript application/javascript
JSON application/json
HTML text/html
XML application/xml

binary

相当于Content-Type:application/octet-stream，只可以上传二进制数据，通常用来上传文件，由于没有键值对，一次只能上传一个文件。

multipart/form-data与x-www-form-urlencoded的区别

前者既可以上传文件等二进制数据，也可以上传表单键值对，只是最后转化为一条信息。

后者则只能上传键值对，并且键值对是分开的。
